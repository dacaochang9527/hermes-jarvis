# 屠龙 D3 分层筛选与降噪监控

## 场景

用于 A 股“屠龙战术”D3 当日观察池：前一交易日 D1 首板，后一交易日 D2 冲高回落、不破 D1 支撑，今天 D3 只做观察与提醒。

## 推荐流程

1. 明确交易日：D1=上上个交易日首板，D2=上个交易日确认，D3=今天观察。遇到周一/节假日时必须按真实交易日回退，不按自然日粗算。
2. 先跑硬条件：D1 首板、排除 ST/退市/不适合标的、D2 量能不过 2 倍、D2 有冲高回落、D2 收盘不破 D1 支撑。
3. 再做软评分：D1 封板时间、炸板次数、封板资金；D2 量比、回落幅度、安全垫、成交额、换手率、是否仍大涨。
4. 分层限额：A 强观察最多 2 只，B 轻观察最多 2 只，C 备选不进高频监控；主监控总数默认不超过 4 只。
5. 更新监控池文件后，立刻用强制运行验证脚本实际读取的是新池，而不是旧硬编码池。

## 分层口径

- A 层：结构标准、D2 量能温和、回落充分、不贴失效位，优先看。
- B 层：硬条件通过但有明显瑕疵，如炸板偏多、尾盘封板、换手偏高、成交额过大、回落刚达标，只轻观察。
- C 层：分数不低但为了降噪不进高频提醒；只有 A/B 失效或盘中结构明显转强时再人工查看。

## 状态表达规范

股票池、提醒和复盘中优先使用 **“日期 + D几”**，周几作为辅助信息，例如：`0526D3（周二）`、`0527D4（周三）`。不要只写“周二D3”，因为跨周后容易混淆；日期+D序号更适合复盘、文件名、监控池流转和历史查询。

推荐写法：

- `0527D3候选区`：表示 5 月 27 日当天正处 D3 观察/低吸窗口；不要再命名为“低吸池”，因为 D2筛选应同时容纳低吸潜质和强势潜质；
- `0526D3走强 → 0527D4强势雷达`：表示 D3 当天涨停、强势未板、强回收或突破前高，次日转入 D4 强势雷达；
- `0526D3未走强 → 0527删除`：表示前一日 D3 没有走强，次日不再留在候选区；
- `0527新D3候选区`：表示 5 月 27 日新生成的当天 D3 票。

D3候选区只放当天 D3。前一交易日已经是 D3 的票，次日就是 D4：若 D3 当天未走强，直接删除；若 D3 当天涨停、强势未板、强回收或突破前高，则转入 D4 强势雷达池，而不是继续 D3 候选区监控。


## D3候选区容量与提醒格式

D3候选区只放当天 D3，但容量不宜过小。实践后默认建议：

- **D3候选区 8–10 只**：从 D1/D2 评分排序取前排，避免 4 只过窄漏掉结构好的票；
- **D2筛选要同时满足两类条件，不是二选一**：D3候选区不是“低吸潜质票 OR 强势潜质票”的并集，而是“低吸可执行性 AND 强势潜质”的交集。也就是说，候选必须既有清楚的买点/止损/回落承接位置，又具备 D3 回收观察价并向上扩展的可能。低吸但没有强势潜质的弱修复票不进；强但没有合理低吸/承接参与点的追高票也不进。
- **高频/强提醒不超过 4–5 只**：只对真正进入买点附近、接近失效、回收观察价、明显异动的票发单股提醒；
- **其余候选区票只进低频快照**：快照内容每 15 分钟生成一次，和发送解耦。事件检测仍保持每 5 分钟，触发买点/止损/参与区时立即提醒；若同一轮同时满足事件和 15 分钟快照，事件优先，只发事件，不叠加快照，避免消息过长和 iLink 限流；同时把该快照保存为“待发送快照”，在事件后的约 3 分钟补发。若 3 分钟后又有新事件，继续事件优先，但待发送快照不能被丢弃，可合并/替换为最新一份 15 分钟快照。

提醒内容要围绕“能不能动手”和“错了怎么办”，不要堆指标。单股提醒推荐 4 行：

```text
0527D3｜600530 交大昂立｜水下观察
现价 7.15｜买点 7.20–7.33｜止损 6.85
动作：只等回到买点区/回收7.31；跌破6.85放弃
风险：距止损 +4.4%｜量能 1.52亿｜14:40
```

字段优先级：

1. 状态：`低吸区` / `水下观察` / `回收观察价` / `接近失效` / `跌破失效` / `强势不追`；
2. 点位：现价、涨跌幅、买点区、观察价/回收价、止损/失效位；
3. 动作：等、可小仓观察、放弃、不追、转强势雷达；
4. 参考压缩到一行：行业/板块、距止损、成交额、时间。

单股提醒推荐 4 行格式：

```text
0527D3｜600530 交大昂立｜食品加工｜水下观察
现价 7.15（-2.19%）｜买点 7.20–7.33｜止损 6.85
动作：只等回到买点区/回收7.31；跌破6.85放弃
参考：距止损+4.4%｜成交额1.52亿｜14:40
```

字段取舍：

- **涨跌幅保留**：它不是买点核心，但能快速感知行情温度，放在现价后括号里；
- **成交额保留，命名为成交额，不叫量能**：成交额用于判断流动性/资金关注度；量能更适合筛选阶段用 D2/D1 量比或量比，不在提醒里重复展示；
- **行业/板块保留**：放在第一行股票名后，帮助判断是否有板块共振。若无法稳定取到板块，则用 D1 涨停池里的所属行业兜底；
- 去掉或降级展示：昨收、今开、最高、最低、安全垫等原始行情字段；除非它们直接影响动作。

快照消息可以更短，只展示“代码｜行业｜状态｜现价(涨跌幅)｜买点/止损｜动作”。


## 强势雷达池指标与参与区触发

强势雷达池和低吸池不是同一套指标。低吸池看“买点/止损”，强势雷达看“强度/晋级/承接/失败线”。但强势雷达不应只发状态提醒后让用户再判断；更好的方式是预先定义 **参与区/触发区**，只有价格进入参与区或触发区附近才提醒。

强势雷达建议保留的核心字段：

- 状态：`参与区` / `接近涨停` / `涨停` / `回踩承接` / `突破确认` / `冲高回落` / `突破失败`；
- 现价 + 涨跌幅；
- 参与区：回踩承接区、突破确认价、接近涨停区、回封确认区；
- 失败线：跌破承接位、突破后跌回、炸板后无法回封；
- 涨停价 / 昨日高点 / 今日高点；
- 成交额；
- 行业/板块；
- 动作：`进入参与区才看` / `突破站稳再看` / `只看回封` / `跌破失败线放弃`。

强势雷达的触发原则：

1. **参与区触发优先于状态触发**：不要只因“涨幅 +5%”就提醒；应在接近涨停、突破前高、回踩承接位、回封确认等可执行位置触发。
2. **提醒要把判断前置**：消息里直接写“参与区”和“失败线”，减少用户收到后再临场判断。
3. **远离参与区只进快照**：如果已经大幅拉离、没有可执行点，只在低频快照中显示，不发单股强提醒。

模板：

```text
0527D4｜605006 山东玻纤｜玻璃玻纤｜接近涨停区
现价 17.70（+8.65%）｜参与区 17.70–17.92｜失败线 17.20
动作：只看冲板/回封确认；不到回封不追半路
参考：涨停17.92｜成交额10.8亿｜14:05
```

```text
0527D4｜603459 红板科技｜元件｜突破确认区
现价 115.40（+6.56%）｜参与区 115.15上方站稳｜失败线 115.15下方
动作：突破前高并站稳才看；跌回前高下方放弃
参考：今日高点116.20｜成交额24.0亿｜13:40
```


## D1/D2 形态质量补充

用户补充的盘面判断应放在 D1/D2 筛选阶段，而不是 D4 强势雷达：

- **D1 炸板后快速回封**：用于评估 D1 首板质量。炸板说明有分歧和抛压，但若能较快回封，说明承接资金强，仍算可接受；若长期不回封或反复烂板，则 D1 质量下降。
- **D2 收盘十字星更好**：用于评估 D2 确认质量。D2 冲高回落后若收成相对平衡的十字星，代表分歧存在但未明显走坏。
- **D2 上引线太长风险大**：如果 D2 冲高回落明显、上影线过长、收盘离高点太远，说明上方抛压重，D3 承压风险升高。

这些都是生成 `0527D3候选区` 时的筛选/评分项，不是 `D4强势雷达` 的监控指标。
