# 屠龙 D3/D4 观察区与持仓区概念模型

## 状态

讨论稿沉淀，适用于后续 A 股屠龙 D3/D4 监控、持仓管理、active watchlist 字段设计。后续会话若继续修订本概念，应更新本文件，并在 `tulong-d3-d4-monitoring.md` 保持摘要同步。

## 核心定稿

### 1. 统一使用“观察”，不再使用“候选”

用户确认：候选和观察含义接近，统一为“观察”。后续用户可见概念不再出现“候选区”。

推荐命名：

```text
0528D3观察区
0528D4观察区
```

工程字段：

```text
pool_type = watch
```

不再使用：

```text
candidate
候选区
```

如果历史文件名或旧文本里出现 `candidate`，只视为旧遗留/生成原因，不作为新概念继续使用。

### 2. D3 需要分区

D3 买入动作会改变提醒目标，因此 D3 也需要分区：

```text
D3观察区：未买入，寻找买点/回收/失效
D3持仓区：已买入，管理当天持仓风险
```

这不是“候选 vs 买入”，而是：

```text
观察 vs 持仓
```

### 3. D4 也分观察区和持仓区

```text
D4观察区：未买入，寻找强势参与区/突破/回踩承接
D4持仓区：已买入，管理次日止损/止盈/减仓/退出
```

原来的“D4强势雷达区”不再作为主命名；它可以理解为 `D4观察区` 的内部提醒逻辑：看强势参与区、突破、回踩承接、失败线。

## D3持仓 vs D4持仓：提醒指标不同

用户确认一个关键差异：

```text
D3持仓：当天买入，T+1 限制下不能卖，只能考虑加仓/风控观察
D4持仓：次日已可卖，可以卖，也可以加
```

因此，`D3持仓区` 和 `D4持仓区` 的提醒指标不能一样。

### D3持仓区提醒目标

D3持仓区的核心不是“卖不卖”，而是：

```text
是否守住买入逻辑
是否允许加仓
是否触发风险标记
尾盘是否需要记录隔夜风险
```

D3持仓区应提醒：

```text
成本价
现价相对成本盈亏
原始买点区/观察价
失效位/防守线
是否回到可加仓区
是否跌破失效位但因 T+1 只能标记风险
尾盘收盘位置：强、弱、接近失效、隔夜风险
```

D3持仓区动作语言：

```text
可加仓观察
不加，持有观察
风险标记，明日优先处理
尾盘弱，记录隔夜风险
```

避免使用：

```text
卖出
减仓
止盈
```

因为 D3 当天买入后不能卖。

### D4持仓区提醒目标

D4持仓区进入可卖日，核心是：

```text
是否卖出/减仓/止损/止盈
是否仍允许加仓
是否转弱退出
```

D4持仓区应提醒：

```text
成本价
现价相对成本盈亏
可卖数量
止损/失败线
止盈/减仓区
是否突破确认
是否回踩承接可加仓
是否跌破失败线应处理
尾盘是否留仓
```

D4持仓区动作语言：

```text
止损/退出
减仓/止盈
继续持有
回踩承接可加仓观察
突破确认后持有或加仓观察
尾盘不强则降低仓位
```

### 对比表

| 阶段 | pool_type | 能否卖 | 能否加 | 主要提醒 |
|---|---|---:|---:|---|
| D3持仓区 | position | 否 | 是 | 成本、防守线、加仓区、隔夜风险 |
| D4持仓区 | position | 是 | 是 | 止损/止盈/减仓、加仓、是否留仓 |

## 工程字段建议

`stage` 表示阶段：

```text
0527D3
0528D4
```

`pool_type` 表示身份：

```text
watch      # 观察区，未买入
position   # 持仓区，已买入
```

可选字段：

```text
entry_date
entry_stage
entry_price
quantity
sellable_quantity   # D3当天买入为0，D4起通常等于可卖持仓
cost_amount
position_status     # open / reduced / closed
source_file
```

## 0527D3 买入后的状态示例

0527 当天买入后：

```text
600863 华能蒙电
stage = 0527D3
pool_type = position
entry_price = 6.170
quantity = 600
sellable_quantity = 0
```

```text
603303 得邦照明
stage = 0527D3
pool_type = position
entry_price = 25.440
quantity = 200
sellable_quantity = 0
```

它们仍然是 `0527D3`，但不再属于 `0527D3观察区`，而是进入 `0527D3持仓区`。

次日 0528：

```text
stage = 0528D4
pool_type = position
sellable_quantity = quantity
```

进入 `0528D4持仓区`，提醒逻辑从“买点提醒/D3不可卖风控”变成“可卖日持仓管理”。
