# 0601-0602D3 共同策略结论

生成时间：2026-06-02 晚间

## 复盘来源

本结论基于以下两份策略复盘：

- `0601D3_strategy_conclusion_20260601.md`
- `0602D3_strategy_conclusion_20260602.md`

0601 更强调 D1/D2/D3 的过滤质量；0602 更暴露单一 D3 watch 池对强势延续票的漏选问题。两天合并后，策略共同点不是“扩大观察池”，而是把 D3 做成分层验证系统。

## 核心共识

```text
D3 不是单纯低吸，也不是单纯追强；
核心是先用 D1/D2 把结构筛干净，
再在 D3 分成“低吸可执行”和“强势延续雷达”两条线分别验证。
```

## 共同策略结论

### 1. D1 是底池，不是买点池

D1 首板只负责产生底池，不能直接等同于 D3 可执行观察。

需要提高权重的 D1 因子：

- 炸板次数；
- 封板干净度；
- 首封/回封质量；
- 成交额和换手拥挤度。

共同判断：D1 不是越热闹越好，而是越干净越好。炸板少、封板结构干净、成交不过度拥挤的票，后续 D3 成功概率更高。

### 2. D2 是关键过滤层

D2 不能只机械判断“冲高回落”，重点是冲高回落之后是否守住结构。

需要强化的 D2 因子：

- 回落后不破 D1 支撑；
- 收盘仍健康，最好接近平衡/十字；
- 避免高开低走、收盘破位、明显出货形态；
- D2 量比 2–3 倍时，需要其他条件补强。

共同判断：D2 是筛掉假强票的关键层。D2 承接不够，D3 不应该承担“救票”功能。

### 3. D3 的关键是确认，不只是低吸

D3 进入买点区只是第一步，更关键的是进入买点区后能否回收观察价。

D3 低吸观察优先级：

1. 进入买点区；
2. 不跌破失效位；
3. 回收观察价；
4. 回收后能站稳或继续承接。

共同判断：只进买点区但不能回收观察价，只能算“可观察”；进入买点区后回收观察价，才是更高质量的确认。

### 4. D3 必须分层，不能只有一个 watch 池

0601 指向“核心观察、强势确认观察、低频备选、剔除”的分层；0602 进一步说明，需要把“低吸观察池”和“强势延续雷达”拆开。

建议后续至少保留两条线：

1. **低吸观察池**
   - 口径：低吸可执行性 AND 强势潜质；
   - 用途：买点区、观察价、失效位、单股提醒；
   - 目标：少噪音、可执行、能复盘。

2. **强势延续雷达**
   - 口径：D1 全量底池中未入 D3 watch，但 D3 当日强势延续；
   - 用途：记录漏选强势样本，复盘板块强弱和规则缺口；
   - 注意：不直接等同于低吸买点，不输出追高动作。

### 5. 减少噪音和避免漏强要分开解决

不能通过单纯扩大 D3 watch 池来解决漏强问题，否则微信提醒会变吵，核心池会变脏。

更好的结构是：

- watch 池继续保持干净，服务可执行提醒；
- 强势雷达单独记录，服务复盘和规则修订；
- 同板块多票时，额外做板块内强弱排序，避免选到相对弱票而漏掉强势延续票。

## 候选规则

后续 D3 生成和复盘建议先按以下候选规则验证：

1. D1 评分必须提高“炸板次数、封板干净度、成交拥挤度”的权重。
2. D2 保留条件必须强调“冲高回落后不破 D1 支撑，收盘仍健康”。
3. D3 低吸观察必须看“进入买点区后是否回收观察价”，不能只看是否跌到低吸区。
4. D3 输出应拆成“低吸观察池”和“强势延续雷达”。
5. 强势延续雷达只用于复盘与漏选样本记录，不直接作为追高买点。

## 结论状态

当前结论是 0601D3 与 0602D3 两天样本共同指向的策略修订候选，不视为最终定稿。后续需要继续用更多交易日样本验证，若新样本冲突，以多日统计结果为准。
