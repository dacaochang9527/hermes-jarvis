# 期货个人盯盘交易 MVP 计划

> 状态：仅落盘计划，暂不执行实现。

**目标：** 做一个面向个人使用的期货快速盯盘与买卖辅助 MVP，先解决“少量品种、关键价位、信号提醒、下单前决策辅助”，不直接做全自动高频下单。

**定位：** 个人半自动交易驾驶舱，而不是机构级高频系统。第一阶段强调稳定、低噪音、可复盘、可人工确认。

**边界：** 本计划只输出信号、提醒、风控与复盘辅助，不构成投资建议，不默认接入实盘自动下单。

---

## 一、MVP 核心思路

### 1. 先做“盯盘辅助”，不要一开始做真高频

个人场景下真正的高频交易门槛很高：低延迟行情、撮合接口、托管机房、交易所规则、滑点、手续费、风控和合规都很重。

MVP 更合理的切入点：

- 只盯 3-10 个期货品种；
- 只做日内短线/分钟级辅助；
- 只提醒“接近可操作区”；
- 人工确认是否下单；
- 所有信号自动记录，方便复盘。

### 2. MVP 应该解决的真实痛点

- 不想一直盯屏，但又怕错过关键点；
- 盘中波动快，人工容易追涨杀跌；
- 需要提前定义入场区、止损线、止盈/移动止盈；
- 需要减少无效提醒，只在状态变化时提醒；
- 需要复盘“提醒是否有效、自己是否执行变形”。

## 二、第一版功能范围

### Task 1: 建立自选合约与关键位配置

**目标：** 用一个本地配置文件维护关注品种、方向偏好、关键价位和风控线。

**建议字段：**

- `symbol`：合约代码；
- `name`：品种名；
- `direction_bias`：long / short / neutral；
- `watch_zone`：观察区间；
- `trigger_price`：触发价；
- `invalid_price`：失效价；
- `take_profit_zone`：止盈参考区；
- `max_loss_per_trade`：单笔最大亏损；
- `note`：人工备注。

**验收：** 能通过配置文件表达 3-10 个合约的盯盘条件。

### Task 2: 接入分钟级行情源

**目标：** 获取关注合约的最新价、涨跌幅、成交量、持仓量等字段。

**优先级：**

1. 先用免费/低成本行情源做原型；
2. 再评估是否接入期货公司 CTP、交易终端导出、第三方行情服务；
3. 不在第一版承诺毫秒级延迟。

**验收：** 能定时拉取自选合约行情，并保存原始快照。

### Task 3: 实现状态机提醒

**目标：** 把合约从“普通观察”推进到“接近参与区 / 已触发 / 失效 / 止盈观察”。

**状态建议：**

- `idle`：未接近；
- `near_watch_zone`：接近观察区；
- `triggered`：触发条件满足；
- `invalidated`：跌破/突破失效条件；
- `profit_watch`：进入止盈/移动止盈区。

**提醒纪律：**

- 只在状态变化时提醒；
- 同一合约同一状态设置冷却时间；
- 提醒里必须带触发价、当前价、失效价、风险距离。

**验收：** 行情变化时能产生清晰、低噪音的提醒事件。

### Task 4: 建立本地事件与交易日志

**目标：** 所有提醒、人工决策、模拟/实盘成交都可追踪。

**日志类型：**

- 行情快照；
- 信号事件；
- 提醒投递记录；
- 人工动作记录；
- 成交/模拟成交记录；
- 收盘复盘记录。

**验收：** 任意一天能回看“系统提醒了什么、自己做了什么、结果如何”。

### Task 5: 做一个最小盯盘输出界面

**目标：** 先不做复杂 App，用命令行/Markdown/微信推送即可。

**第一版输出：**

- 当前关注合约列表；
- 每个合约当前状态；
- 参与区、失效线、当前价；
- 最新提醒事件；
- 当日已触发/已失效列表。

**验收：** 手机微信里能看懂当前是否需要盯盘。

### Task 6: 增加日终复盘

**目标：** 每天收盘后生成一份复盘，判断规则是否有效。

**复盘字段：**

- 今日触发次数；
- 有效提醒数量；
- 误报数量；
- 错过机会；
- 最大浮盈/浮亏；
- 是否执行纪律；
- 次日需要关注的价位。

**验收：** 能用复盘更新第二天的关键位配置。

## 四、分阶段路线

### Phase 1: 纸面 MVP

- 建 watchlist 配置；
- 手动录入 3-5 个合约关键价位；
- 拉取或手动导入行情；
- 生成提醒事件；
- 微信/本地输出。

### Phase 2: 半自动盯盘

- 定时行情拉取；
- 状态机去重提醒；
- 本地日志落盘；
- 日终复盘报告。

### Phase 3: 交易辅助

- 增加模拟持仓；
- 计算仓位、止损金额、盈亏比；
- 下单前生成确认卡片；
- 不自动下单，只辅助决策。

### Phase 4: 可选实盘接口

- 评估 CTP/期货公司 API；
- 加入权限、风控、熔断；
- 先做模拟盘/仿真盘；
- 再考虑有限实盘自动化。

## 六、暂缓事项

第一版暂不做：

- 真正毫秒级高频；
- 自动实盘下单；
- 全市场扫描；
- 复杂机器学习预测；
- 多账户资金管理；
- 大规模参数优化。

先把“个人少量合约盯盘 + 低噪音提醒 + 纪律复盘”跑顺。
